# Project Notes

## Purpose
This project was created to practice MySQL database design, data loading, joins, aggregation, filtering, and reporting queries using a simulated retail sales dataset.

## Data Model
The database uses four connected tables:

- `customers`
- `products`
- `orders`
- `payments`

Each order connects a customer to a product, and each payment connects back to an order.

## Analysis Focus
The SQL queries explore:

- Customer types and locations
- Product categories and pricing
- Order quantities and order statuses
- Payment completion and refunds
- Revenue by product and category
- Customers with the highest paid spending
- Pending transactions that may require follow-up

## Portfolio Note
All data in this project is simulated and used for learning and portfolio purposes.
